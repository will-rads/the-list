## Wire (server writes are SECURITY DEFINER RPCs — supabase.rpc())
1. **Auth in the existing onboarding** (keep every screen/step): signInWithOtp({email,
   options:{data:{full_name, ig_handle}}}) → verifyOtp (type 'email', 6-digit code). The step that
   currently collects phone becomes an email input (minimal label edit; phone OTP later).
   "I have an invite" path: after auth, rpc `redeem_invite(p_code)`. After login fetch own
   `profiles` row: status 'pending' → simple full-screen "You're on the queue — we review every
   application" (reuse onboarding classes, no new design); 'approved' → app.
2. **Live data when logged in**: events = from('events').select('*, venues(name,area)')
   .eq('status','published'); my applications = from('applications').select('*, events(*)')
   .eq('member_id', uid); saves; notifications ordered desc + realtime channel (postgres_changes
   INSERT, filter user_id=uid) prepending live; profile = own profiles row (creator_data jsonb =
   SEED_PROFILE shape). Keep SEED_* as logged-out fallback so intro + onboarding render pre-auth.
3. **Buttons → RPCs**: apply = `apply_to_event(p_event)` (returns 'applied'|'waitlist' — reflect);
   cancel = `cancel_application(p_app)`; confirm = `confirm_pick(p_app)`; decline =
   `decline_pick(p_app)`; save = `toggle_save(p_event)`; closing the notifications sheet marks own
   unread read via update (RLS allows).
4. **Status mapping**: applied/waitlist/picked/confirmed/declined/expired/not_selected/checked_in/
   no_show/cancelled → existing GS states (not_selected→notSelected, checked_in→checkedIn, etc).
   Story: stories row per application (verdict pending + due_at → due; verified → verified with
   {score, reason}).
5. **Signup extras**: after first login call supabase.functions.invoke('creator-data',
   {body:{handle: ig_handle}}) and save result to own profiles.creator_data via update. Profile
   screen reads creator_data when present.
6. **Share sheet**: action "dm" → label "WhatsApp", new inline "whatsapp" glyph in HICONS (same
   custom-SVG approach as instagram), opens https://wa.me/?text=<encoded link> where link =
   https://the-list-omega.vercel.app/v3/e?id=<event.id>. "Copy link" copies that same link
   (navigator.clipboard). To Story + More stay toast-only.
